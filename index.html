Глоссарий для чайника

все термины из этого документа простыми словами
Читай эту секцию первой, если какое-то слово в роадмапе непонятно. Возвращайся сюда в любой момент за 3,5 года — тут расшифровано всё, что встречается ниже.

Учёба / документы

ECTS
Европейская система зачётных единиц. 1 ECTS ≈ 25-30 часов работы студента (лекции + самостоятельная работа). Семестр = обычно 30 ECTS.
E (Egzamin)
Предмет заканчивается экзаменом — обычно письменный тест/задача в сессию.
O (Ocena / zaliczenie na ocenę)
Зачёт с оценкой — без формального экзамена, но с итоговой оценкой по работе в семестре.
ZAL (Zaliczenie)
Простой зачёт (сдал/не сдал), без оценки в диплом.
Praktyka zawodowa
Обязательная производственная практика — стажировка в компании, засчитывается в ECTS.
Proseminarium / Seminarium dyplomowe
Занятия, где готовишь и защищаешь дипломную работу поэтапно.
Wybieralny przedmiot
Предмет по выбору — из списка нескольких выбираешь один.

Облако (Cloud)

Cloud Computing
Аренда вычислительных мощностей (серверов, хранилищ, баз данных) у провайдера (AWS, Azure, GCP) через интернет вместо покупки своего железа.
AWS / Azure / GCP
Три крупнейших облачных провайдера: Amazon Web Services, Microsoft Azure, Google Cloud Platform.
Виртуализация
Технология, позволяющая на одном физическом сервере запускать несколько независимых "виртуальных" компьютеров.
Контейнеризация
Упаковка приложения со всем нужным окружением в лёгкий изолированный "контейнер", который одинаково работает везде.
Docker
Самый популярный инструмент контейнеризации.
Kubernetes (K8s)
Система, которая автоматически управляет множеством контейнеров: запускает, масштабирует, перезапускает при сбое.
IaC (Infrastructure as Code)
Описание серверов и инфраструктуры кодом (файлами), а не ручными кликами в панели управления.
Terraform
Популярный инструмент для IaC — один и тот же код разворачивает инфраструктуру в AWS, Azure и др.
CI/CD
Continuous Integration / Continuous Deployment — автоматическая проверка и доставка кода в продакшн при каждом изменении.

Разработка

Front-end
Та часть приложения, которую видит и с которой взаимодействует пользователь (сайт, кнопки, интерфейс).
Back-end
"Невидимая" серверная часть: логика, база данных, обработка запросов.
API
Application Programming Interface — правила, по которым программы обмениваются данными друг с другом.
Git / GitHub
Git — система контроля версий кода (история изменений). GitHub — сайт для хранения Git-репозиториев и совместной работы.
IDE
Integrated Development Environment — программа-редактор с подсветкой кода, автодополнением, отладкой (VS Code, PyCharm, IntelliJ).
ООП (OOП)
Объектно-ориентированное программирование — стиль написания кода через объекты с свойствами и поведением.
NoCode / LowCode
Создание приложений почти без написания кода, через визуальные конструкторы.
SQL / БД
Structured Query Language — язык запросов к реляционным базам данных (таблицы, строки, столбцы).

Сети и безопасность

SSH
Secure Shell — защищённый способ подключиться к удалённому серверу через терминал.
VPN
Виртуальная частная сеть — зашифрованное соединение поверх обычного интернета.
Firewall
"Файрвол" — фильтр сетевого трафика, блокирует нежелательные подключения.
Шифрование
Преобразование данных так, что прочитать их может только тот, у кого есть ключ.
Мониторинг
Постоянное отслеживание состояния серверов/приложений (нагрузка, ошибки, доступность).
Blockchain
Распределённая база данных, где записи (блоки) нельзя незаметно изменить задним числом.

Работа / карьера

Junior / Mid / Senior
Уровни опыта разработчика: начинающий → уверенный специалист → эксперт, ведущий проекты.
Сертификация (AWS/Azure/K8s)
Официальный экзамен от вендора, подтверждающий знания — сильно повышает шансы на работу.
RKM
Rodzinny Kredyt Mieszkaniowy — польская госпрограмма ипотеки без первого взноса (гарантия от BGK).

★ Architektura Cloud Computing (Sem 4)

Учит, как вообще устроено облако: типы сервисов (IaaS/PaaS/SaaS), регионы и зоны доступности, как считать стоимость, как проектировать отказоустойчивую систему.

Это фундамент всей специализации — без этого остальные ★-курсы будут непонятны.

Виртуальные машины vs контейнеры, Docker: образы, контейнеры, Dockerfile, docker-compose.

Docker — обязательный навык на 90% вакансий Cloud/DevOps.

HTML/CSS/JavaScript + фреймворк (React/Vue) — интерфейсы, которые видит пользователь.

★ Platformy i usługi chmur obliczeniowych (Sem 6)

Конкретные сервисы AWS/Azure: compute (EC2/VM), storage (S3/Blob), сети (VPC), базы данных (RDS).

★ Programowanie Back-end (Sem 6)

Серверная логика, REST API, работа с базой данных со стороны сервера (Node.js/PHP/Python).

Облачный инженер часто разворачивает и настраивает именно back-end приложения.

★ Architektura i komunikacja między systemami (Sem 6)

Как отдельные сервисы общаются: REST, message queues (Kafka/RabbitMQ), микросервисы vs монолит.

Ключевая тема для собеседований на позиции Cloud/Backend Engineer.

microservices.io YouTube: очереди сообщений

Распределённые вычисления: как множество серверов работают как единая система + основы блокчейна.

Даёт понимание отказоустойчивости и консенсуса — применимо и вне блокчейна (Kubernetes, распределённые БД).

YouTube: Distributed Systems

🐍 Python

Отступы вместо { } — код внутри блока сдвинут пробелами.

# переменная и типы
name = "Alex"
age = 20
is_student = True

# условие
if age >= 18:
    print("совершеннолетний")
else:
    print("несовершеннолетний")

# цикл
for i in range(5):
    print(i)

# функция
def greet(name):
    return f"Привет, {name}!"

# класс (ООП)
class Student:
    def __init__(self, name):
        self.name = name
    def say_hi(self):
        print(f"Я {self.name}")

☕ Java

Строго типизирован, всё внутри классов, блоки в { }.

// переменная

🗄️ SQL

Язык запросов к базе данных — не программирование в обычном смысле, а декларативные команды.

-- создание таблицы
CREATE TABLE students (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);

-- вставка данных
INSERT INTO students (id, name, age) VALUES (1, 'Alex', 20);

-- выборка
SELECT name, age FROM students WHERE age >= 18 ORDER BY name;

-- изменение
UPDATE students SET age = 21 WHERE id = 1;

-- соединение таблиц
SELECT s.name, c.title FROM students s
JOIN courses c ON s.id = c.student_id;

Работа с ИИ на учёбе

как использовать Claude/ChatGPT на каждом этапе
Главное правило: ИИ — это инструмент для понимания и ускорения, а не замена собственных знаний. На экзаменах и защите диплома спрашивают тебя, а не ИИ. Используй его, чтобы разбираться быстрее и глубже, а не чтобы не думать.
01

Разбор непонятной темы с лекции

Если что-то на лекции осталось непонятным — попроси объяснить простыми словами и с примером, прежде чем читать учебник.

Промпт: "Объясни [тема] простыми словами, как чайнику. Дай короткий пример из реальной жизни."
02

Написание и отладка кода

ИИ хорош как "напарник": объясняет ошибки, предлагает варианты решения. Но пиши код сам, а не копируй бездумно — иначе не сдашь устную часть/защиту проекта.

Промпт: "Вот код и ошибка: [...]. Объясни, почему она возникает, но не переписывай всё за меня — подскажи, что исправить."
03

Подготовка к экзаменам

Попроси ИИ составить список вопросов по теме и проверять твои ответы, как экзаменатор — это активное повторение, которое запоминается лучше пассивного чтения.

Промпт: "Составь 10 вопросов по теме [X] уровня экзамена в университете и по очереди проверяй мои ответы, указывая на ошибки."
04

Дипломный проект и семинары

ИИ полезен для структурирования работы, поиска источников по теме, проверки логики текста. Итоговый текст и идею должен формулировать ты — руководитель и комиссия быстро видят разницу.

Промпт: "Вот план моей главы диплома: [...]. Проверь логику и структуру, укажи, где не хватает аргументов."
05

Английский язык

Практикуй письмо и разговор с ИИ: проси исправлять ошибки, объяснять грамматику, вести диалог на IT-темы.

Промпт: "Побудь моим собеседником для практики английского на IT-темы. Исправляй мои ошибки после каждой моей реплики."
06

Практика (Praktyka zawodowa) и облачные задачи

На практике в компании ИИ ускоряет чтение чужого кода/документации и написание конфигов (Terraform, YAML, Dockerfile) — но всегда проверяй, что оно реально делает, прежде чем запускать на реальном сервере.

Промпт: "Объясни построчно, что делает этот Terraform/YAML файл, и какие у него могут быть побочные эффекты."
Инструменты: Claude и ChatGPT — для объяснений и текста. GitHub Copilot — автодополнение кода прямо в редакторе (у студентов есть бесплатный доступ через GitHub Student Developer Pack).

Установка с нуля — по шагам

конкретные команды, не просто ссылки
Порядок ниже — с первой недели учёбы. Не обязательно ставить всё сразу: Git + VS Code + Python нужны с первого дня, остальное — ближе к семестру, где оно появляется в таблице выше.
01Git + GitHub + SSH-ключ
›
  1. Скачай и установи Git: git-scm.com/downloads (на Windows на всех экранах установщика можно жать «Next» с настройками по умолчанию).
  2. Открой терминал (на Windows — Git Bash, на Mac/Linux — обычный терминал) и представься Git:
git config --global user.name "Твоё Имя"
git config --global user.email "твоя_почта@example.com"
  1. Создай SSH-ключ, чтобы не вводить пароль каждый раз:
ssh-keygen -t ed25519 -C "твоя_почта@example.com"
# на все вопросы просто жми Enter (оставить по умолчанию)
  1. Скопируй публичный ключ:
cat ~/.ssh/id_ed25519.pub
  1. На github.com/settings/keys → «New SSH key» → вставь скопированное → Save.
  2. Проверь, что всё работает:
ssh -T git@github.com
# должно ответить: "Hi username! You've successfully authenticated"
02VS Code + расширения
›
  1. Скачай с code.visualstudio.com, установи как обычную программу.
  2. Открой вкладку расширений: Ctrl+Shift+X (Mac: Cmd+Shift+X).
  3. Найди и нажми «Install» на каждом: Python, GitLens, Docker, YAML, Live Server, Prettier.
  4. Встроенный терминал открывается сочетанием Ctrl+` — все команды из этого гайда можно вводить прямо туда.
03Python
›
  1. Скачай на python.org/downloads (последняя версия 3.x).
  2. Windows: на первом экране установщика обязательно поставь галочку «Add python.exe to PATH» — без неё команда python не будет работать в терминале.
  3. Проверь установку:
python --version
pip --version
  1. Для учебных проектов создавай отдельное окружение (venv), чтобы библиотеки разных проектов не конфликтовали:
python -m venv venv
# активация на Windows:
venv\Scripts\activate
# активация на Mac/Linux:
source venv/bin/activate
04Linux (WSL2, если у тебя Windows)
›
  1. Открой PowerShell от имени администратора (правый клик → «Запуск от имени администратора»).
  2. Введи одну команду — она сама установит Linux (Ubuntu) внутри Windows:
wsl --install
  1. Перезагрузи компьютер, когда попросит.
  2. После перезагрузки откроется окно Ubuntu — придумай себе username и пароль (пароль не будет отображаться при вводе, это нормально).
  3. Проверь версию:
wsl --list --verbose

Альтернатива без WSL2 — VirtualBox с образом Ubuntu, если хочешь полностью отдельную виртуальную машину.

05Docker Desktop
›
  1. Скачай на docker.com/products/docker-desktop.
  2. Установи, перезагрузи компьютер.
  3. На Windows при первом запуске выбери backend WSL2 (не Hyper-V), если он предложит.
  4. Открой Docker Desktop — жди, пока в углу появится «Docker is running».
  5. Проверь в терминале, что всё работает:
docker run hello-world

Если видишь текст "Hello from Docker!" — всё установлено правильно.

06AWS Free Tier — аккаунт безопасно
›
  1. Регистрация на aws.amazon.com/free — нужна карта (для верификации, спишется символическая сумма и вернётся).
  2. Сразу после регистрации, до всего остального: зайди в Billing → Budgets → Create a budget → «Zero spend budget» — это бесплатное оповещение на почту, если вообще возникнет любой платёж. Это главная защита от неожиданных счетов.
  3. Включи двухфакторную аутентификацию (MFA) на root-аккаунте: Security credentials → Multi-factor authentication.
  4. Не работай под root каждый день — создай отдельного IAM-пользователя для себя: IAM → Users → Add user, с правами AdministratorAccess для учёбы.
  5. Заходи в консоль дальше уже под этим IAM-пользователем, не под root.
⚠️ Free Tier даёт лимиты (например, 750 часов EC2 t2.micro в месяц). Превышение лимита = реальный платёж. Budget-алерт из шага 2 — не опция, а обязательный шаг.
